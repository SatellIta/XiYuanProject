## API 文档

本文档概述了用于与后端服务交互的主要 API 端点。

### 对话控制器 (`/chats`)

处理所有与对话和疗程相关的操作。

| 方法   | 端点                      | 描述                                                     | 请求体 / 参数                                                              | 成功响应示例                                                                 |
| :----- | :------------------------ | :------------------------------------------------------- | :------------------------------------------------------------------------- | :--------------------------------------------------------------------------- |
| `POST` | `/chats/load`             | 从存档文件加载游戏，并获取当前疗程的 AI 初始消息。       | `{ "chatId": "string", "message"?: "string" }`                             | `"你好，我们继续上次的话题吧。"` (AI 的回复)                               |
| `POST` | `/chats/continue`         | 使用新的用户消息继续现有对话。                           | `{ "chatId": "string", "body": "string" }`                                 | `"听起来这让你很困扰。"` (AI 的回复) 或一个 JSON 对象 (见下文)              |
| `POST` | `/chats/confirm-stage`    | 确认问题或解决方案，以过渡到下一个治疗阶段。             | `{ "chatId": "string", "problem"?: "string", "solution"?: "string" }`      | `{ "success": true, "message": "问题已确认，第一疗程结束。" }`             |
| `POST` | `/chats/reset-session`    | 将用户进度重置到指定疗程的开始。                         | `{ "chatId": "string", "sessionNumber": "number" }`                        | `{ "success": true, "message": "已成功重置到疗程 1。" }`                   |
| `GET`  | `/chats/history/:chatId`  | 检索指定对话的完整聊天记录。                             | `chatId` (URL 参数)                                                        | `[{ "role": "user", "content": "你好" }, ...]` (聊天记录数组)              |
| `GET`  | `/chats/history`          | 列出 Redis 中所有活跃对话的 ID。                         | -                                                                          | `["chatId1", "chatId2"]` (对话 ID 数组)                                    |
| `POST` | `/chats/end`              | 从 Redis 中删除指定对话的聊天记录。                      | `{ "chatId": "string" }`                                                   | `"对话chatId1已结束, 聊天记录已删除"`                                        |

### 存档控制器 (`/saves`)

管理用户进度的创建、保存和加载。

| 方法   | 端点                  | 描述                               | 请求体 / 参数            | 成功响应示例                                                                 |
| :----- | :-------------------- | :--------------------------------- | :----------------------- | :--------------------------------------------------------------------------- |
| `POST` | `/saves/new`          | 为用户创建一个新的空存档文件。     | `{ "chatId": "string" }` | `"新存档创建成功！"`                                                         |
| `POST` | `/saves/save`         | 保存用户会话的当前状态。           | `{ "chatId": "string" }` | `"存档已成功保存！"`                                                         |
| `GET`  | `/saves/load/:chatId` | 检索指定用户的完整存档数据。       | `chatId` (URL 参数)      | `{ "chatId": "...", "completedTherapySession": 0, ... }` (SaveData 对象) |

### 提示词控制器 (`/prompts`)

管理 AI 使用的系统提示词。

| 方法   | 端点               | 描述                               | 请求体 / 参数                                        | 成功响应示例                                                                 |
| :----- | :----------------- | :--------------------------------- | :--------------------------------------------------- | :--------------------------------------------------------------------------- |
| `GET`  | `/prompts/list`    | 检索所有可用的系统提示词。         | -                                                    | `{ "prompt_default": "你是一个AI助手...", "session_1": "..." }`            |
| `GET`  | `/prompts/:name`   | 按名称检索特定的提示词。           | `name` (URL 参数)                                    | `"你是一个AI助手..."` (提示词字符串)                                       |
| `POST` | `/prompts/set`     | 设置或更新一个系统提示词。         | `{ "systemPromptName": "string", "prompt": "string" }` | `"名称为 my_prompt 的系统提示词已成功设置/更新！"`                         |
| `POST` | `/prompts/delete`  | 按名称删除一个系统提示词。         | `{ "systemPromptName": "string" }`                   | `"名称为 my_prompt 的系统提示词已成功删除！"`                              |

### AI JSON 响应格式

在某些疗程阶段，AI 被指示以特定的 JSON 格式回复，以便客户端可以解析并执行相应操作。

#### Session 1: 问题识别

AI 在每次对话都会返回如下格式的 JSON ， AI 还没发现问题时 problem 的值是 null。

```json
{
  "dialogue": "请问最近你遇到什么麻烦了...",
  "problem": "用户识别出的核心问题描述"
}
```

#### Session 2: 解决方案头脑风暴

在与用户讨论后，当 AI 准备好提出具体的解决方案建议时，它将返回包含对话和两个方案的 JSON。

```json
{
  "dialogue": "这是我为你建议的两个方案...",
  "solution_1": "第一个解决方案的描述",
  "solution_2": "第二个解决方案的描述"
}
```

#### Session 3: 关卡推荐

当 AI 准备好根据用户的解决方案推荐一个具体的治疗“关卡”（例如，一个练习或活动）时，它将返回以下格式的 JSON。

```json
{
  "dialogue": "根据你的选择，我推荐我们进行这个练习...",
  "recommended_level": "推荐的关卡名称"
}
```
